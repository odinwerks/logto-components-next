### 2. Add permission validation utility

**File:** `app/logto-kit/logic/utils.ts`

Add function to validate organization token scopes:

```typescript
export async function validateOrganizationPermissions(
  orgToken: string,
  requiredPermissions: string[],
  requireAll: boolean = true
): Promise<boolean> {
  try {
    // Decode JWT to get scopes (or use introspection)
    const parts = orgToken.split('.');
    if (parts.length !== 3) return false;

    const payload = JSON.parse(Buffer.from(parts[1], 'base64').toString());
    const tokenScopes = payload.scope?.split(' ') || [];

    if (requireAll) {
      return requiredPermissions.every(perm => tokenScopes.includes(perm));
    } else {
      return requiredPermissions.some(perm => tokenScopes.includes(perm));
    }
  } catch (error) {
    console.error('[validateOrganizationPermissions] Failed:', error);
    return false;
  }
}
```

### 5. Update `validateRbac` to check permissions instead of roles

**File:** `app/logto-kit/custom-actions/validation.ts`

Change from role-based to permission-based validation:

```typescript
export async function validateRbac(
  userId: string,
  userData: RbacUserData,
  options: { perm?: string | string[]; requireAll?: boolean } = {} // Changed from 'role' to 'perm'
): Promise<RbacValidationResult> {
  const { perm, requireAll = true } = options;

  if (!perm) return { ok: true };

  // Must have active organization for permission checks
  if (!userData.asOrg) {
    return { ok: false, error: 'NO_ORG_SELECTED', detail: 'Organization permissions require active organization' };
  }

  // Validate org membership
  const orgValidation = await validateOrgMembership(userData.organizations, userData.asOrg);
  if (!orgValidation.ok) return orgValidation;

  // Check permissions in organization token
  if (!userData.organizationPermissions) {
    return { ok: false, error: 'PERMISSION_DENIED', detail: 'No organization permissions available' };
  }

  const requiredPerms = Array.isArray(perm) ? perm : [perm];
  const hasPermission = requireAll
    ? requiredPerms.every(p => userData.organizationPermissions!.includes(p))
    : requiredPerms.some(p => userData.organizationPermissions!.includes(p));

  if (!hasPermission) {
    const detail = `Missing permission(s): ${requiredPerms.join(', ')}`;
    return { ok: false, error: 'PERMISSION_DENIED', detail };
  }

  return { ok: true };
}
```

## How the New Permission System Works

**Before (❌ Custom RBAC):**
- Used hardcoded role mappings (`"president": "role-president-demo-id"`)
- Fetched role names via Management API with M2M token
- Checked role names against custom permission maps

**After (✅ Logto Organization Permissions):**
- Permissions (`"kidnap:kids"`, `"steal:taxes"`) defined in Logto Console organization template
- Roles assigned permissions in Logto Console
- Organization tokens contain permission scopes as JWT claims
- Backend validates permission scopes from organization tokens

**Flow:**
1. Backend calls Management API: `GET /api/organizations/{orgId}/users/{userId}/scopes`
2. Returns array of permission strings like `["kidnap:kids", "steal:taxes"]`
3. `<Protected perm="kidnap:kids">` checks if user's permissions include the required permission
4. Protected Actions API validates Management API response against required permissions

**Required Setup in Logto Console:**
1. **Organization Template** → **Organization permissions** → Add `kidnap:kids`, `steal:taxes`
2. **Organization Template** → **Organization roles** → Create "president" role
3. **Organization Template** → Assign permissions to "president" role
4. **Organization members** → Assign "president" role to users
5. **Application scopes** → Include `organizations`, `organization_roles`
