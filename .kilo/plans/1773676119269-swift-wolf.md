# Plan: Dashboard Modal in LogtoProvider

## Goal
Add dashboard modal state to LogtoProvider so it's seamless — any UserButton in the tree can trigger it.

## The Key Insight
LogtoProvider is a Client Component. Dashboard is an async Server Component. The Server Component (page.tsx) renders Dashboard and passes it as a prop. LogtoProvider just toggles visibility via a boolean.

```
page.tsx (Server Component)
  <LogtoProvider userData={...} accessToken={...} dashboard={<Dashboard />}>
    <DemoApp />
  </LogtoProvider>
```

Dashboard is rendered server-side. LogtoProvider receives it as a ReactNode. It stays hidden until `openDashboard()` is called.

## Changes

### 1. Modify `app/logto-kit/components/handlers/logto-provider.tsx`
- Add `dashboard?: ReactNode` to `LogtoProviderProps`
- Add `isOpen` state, `openDashboard()`, `closeDashboard()` functions
- Extend `LogtoContextValue` with `openDashboard`, `closeDashboard`
- When `isOpen && dashboard`, render overlay with close button + dashboard content
- Overlay: fixed, full screen, backdrop blur, z-index 9999

### 2. Modify `app/page.tsx`
- Fetch user data server-side (like the original Dashboard did)
- Render LogtoProvider with userData, accessToken, and dashboard prop
- Dashboard is rendered as a Server Component child passed to the provider

### 3. Modify `app/demo/Sidebar.tsx`
- Import `useLogto` from logto-kit
- UserButton `do` prop: `() => modal.openDashboard()`

### 4. Revert `app/logto-kit/components/userbutton/index.tsx`
- Remove lazy/Suspense additions
- Back to original import (no Dashboard import needed for modal)
- DashboardModal stays as-is (works in full app context)

## Architecture
```
page.tsx
  ├─ fetchDashboardData() [Server Action]
  └─ <LogtoProvider dashboard={<Dashboard />} userData={...} accessToken={...}>
       └─ <DemoApp> [Client Component]
            └─ <Sidebar>
                 └─ <UserButton do={() => useLogto().openDashboard()} />
```
